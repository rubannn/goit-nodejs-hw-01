# goit-nodejs-hw-01


### Загальний опис застосунку

У кожного з нас директорія Downloads перетворюється на справжній хаос.

Сотні файлів з незрозумілими назвами, дублікати однакових документів, старі файли, які вже давно не потрібні — все це робить пошук потрібного файлу справжнім випробуванням.

Цю проблему можна вирішити за допомогою автоматизації, створивши CLI-застосунок на Node.js, який допоможе проаналізувати стан директорії, знайти дублікати, організувати файли по категоріях та видалити застарілі дані.

Ваше завдання — створити повнофункціональний CLI-інструмент `file-organizer`, який демонструє практичне застосування всіх концепцій, вивчених у першому модулі курсу.

#### Що потрібно реалізувати:

1. Команду `scan` — рекурсивне сканування директорії та збір детальної статистики про файли (кількість, розміри, типи, вік)
2. Команду `duplicates` — пошук файлів з однаковим вмістом через обчислення SHA-256 хешів
3. Команду `organize` — копіювання та сортування файлів по категоріях (Documents, Images, Archives, Code, Videos, Other)
4. Команду `cleanup` — пошук та видалення файлів старіших за вказану кількість днів з підтримкою режиму попереднього перегляду (dry run)
5. Архітектуру на основі `EventEmitter` — кожна команда як окремий клас з подіями для відстеження прогресу
6. Обробку помилок через `try...catch` для всіх файлових операцій з конкретними повідомленнями для різних кодів помилок
7. Використання `streams` для ефективної роботи з великими файлами (≥10 MB) замість fs.readFile() / fs.copyFile()

### Команда scan для аналізу директорії

Команда викликається наступним чином:
```
node file-organizer.js scan /path/to/directory
```

Команда `scan` рекурсивно обходить вказану директорію та всі її піддиректорії, збираючи повну статистику про знайдені файли. Для кожного файлу програма отримує інформацію про його розмір та дату останньої модифікації. Після завершення сканування виводиться детальний звіт у консоль.

Звіт має включати загальну кількість знайдених файлів та їх сумарний розмір.

Далі файли групуються за типом (розширенням), і для кожного типу показується кількість файлів та сумарний розмір. Також відображається розподіл файлів за віком: скільки файлів було змінено за останні 7 днів, за останні 30 днів, та скільки файлів старіші за 90 днів.

Окремо виводяться топ-3 найбільших файли з їхніми розмірами та інформація про найстаріший файл у директорії.

#### Технічна реалізація

Для сканування директорії використовуйте метод fs.readdir() з опцією recursive: true, яка автоматично обходить всі піддиректорії. Якщо ви хочете більше контролю над процесом, можете реалізувати власну рекурсивну функцію, яка для кожної знайденої піддиректорії викликає саму себе.

Для кожного знайденого файлу необхідно викликати fs.stat(), щоб отримати об'єкт статистики. З цього об'єкта беруться дві критично важливі властивості: size (розмір файлу в байтах) та mtime (час останньої модифікації). Щоб визначити тип файлу, використовуйте path.extname(), який витягує розширення з повного шляху до файлу.

Для показу прогресу сканування створіть клас Scanner, який наслідується від EventEmitter. Під час обробки кожного файлу викликайте this.emit('file-found', fileData), де fileData містить інформацію про знайдений файл. По завершенні сканування викликайте this.emit('scan-complete', statistics) з підсумковою статистикою. Це дозволить основній частині програми оновлювати прогрес-бар та показувати користувачу, скільки файлів вже оброблено.

Для групування даних використовуйте Map або звичайні JavaScript об'єкти. Наприклад, створіть `Map<extension, {count, totalSize}>` для збереження статистики по типах файлів. Це дозволить легко підраховувати кількість файлів кожного типу та їхній сумарний розмір.

### Команда duplicates для пошуку дублікатів

Команда викликається наступним чином:
```
node file-organizer.js duplicates /path/to/directory
```
Команда `duplicates` знаходить файли з абсолютно однаковим вмістом, навіть якщо їхні назви різні. Це можливо завдяки обчисленню криптографічного хешу SHA-256 для кожного файлу.

Файли з однаковим хешем мають ідентичний вміст, тому вони є дублікатами.

Після завершення сканування програма групує знайдені дублікати та показує, скільки місця на диску марнується через їх зберігання. Для кожної групи дублікатів виводиться хеш файлу, список усіх копій з їхніми повними шляхами, та підрахунок "витраченого" місця.

Витрачене місце розраховується як розмір файлу, помножений на кількість зайвих копій (загальна кількість мінус один оригінал).

#### Технічна реалізація

Для обчислення хешу використовуйте модуль `crypto` та його метод `createHash('sha256')`. Критично важливо не використовувати fs.readFile() для читання файлів, бо великі файли не поміщаються в оперативну пам'ять.

Замість цього використовуйте `fs.createReadStream()`, який читає файл невеликими порціями (chunks).

Ось як працює обчислення хешу через stream. Спочатку ви створюєте об'єкт хешу через `crypto.createHash('sha256')`. Далі створюєте readable stream для файлу. Кожного разу, коли stream читає черговий chunk даних, ви передаєте його в хеш через метод `update()`.

Коли stream досягає кінця файлу, спрацьовує подія `'end'`, і ви можете отримати фінальний хеш через метод `digest('hex')`, який повертає хеш у вигляді hex-рядка.

Приклад функції для обчислення хешу:
```js
import crypto from 'crypto';
import fs from 'fs';

function calculateHash(filePath) {
  return new Promise((resolve, reject) => {
    const hash = crypto.createHash('sha256');
    const stream = fs.createReadStream(filePath);

    stream.on('data', (chunk) => hash.update(chunk));
    stream.on('end', () => resolve(hash.digest('hex')));
    stream.on('error', reject);
  });
}
```
Функція повертає `Promise`, бо процес обчислення хешу асинхронний. Ви підписуєтеся на події stream: 'data' викликається для кожного chunk (і ми оновлюємо хеш), 'end' означає успішне завершення (і ми повертаємо фінальний хеш), 'error' означає помилку (і ми її відхиляємо через reject).

Після обчислення хешів для всіх файлів згрупуйте їх за значенням хешу. Використовуйте структуру даних `Map<hash, Array<filePath>>`. Якщо масив шляхів для певного хешу містить більше одного елемента, це означає, що знайдено дублікати. Показуйте тільки ті групи, де `filePaths.length > 1`.

Реалізуйте клас `DuplicateFinder extends EventEmitter` з подіями `'file-processed'` для оновлення прогресу обчислення хешів та `'duplicates-found'` для показу фінальних результатів.

### Команда organize для сортування файлів по категоріях

Команда викликається наступним чином:
```
node file-organizer.js organize /source/directory --output /target/directory
```
Команда `organize` копіює файли з вихідної директорії в цільову, сортуючи їх по категоріях в окремі директорії. Категорії визначаються на основі розширення файлу.

Важливо, що оригінальні файли зберігаються на своїх місцях — команда не видаляє нічого з source directory.

У цільовій директорії створюються директорії для кожної категорії: Documents, Images, Archives, Code, Videos та Other. Файли з невідомими розширеннями потрапляють у категорію Other.

Після завершення копіювання виводиться підсумкова статистика про кількість файлів у кожній категорії.

#### Технічна реалізація

Спочатку визначте mapping розширень до категорій.

Створіть об'єкт, де ключі — це назви категорій, а значення — масиви розширень:
```js
const categories = {
  Documents: ['.pdf', '.docx', '.doc', '.txt', '.md', '.xlsx', '.pptx'],
  Images: ['.png', '.jpg', '.jpeg', '.gif', '.svg', '.webp', '.bmp'],
  Archives: ['.zip', '.rar', '.tar', '.gz', '.7z'],
  Code: ['.js', '.py', '.java', '.cpp', '.html', '.css', '.json'],
  Videos: ['.mp4', '.avi', '.mkv', '.mov', '.webm'],
  Other: []
};
```

Категорія Other призначена для всіх нерозпізнаних розширень.

Коли ви визначаєте категорію для файлу, витягуєте його розширення через path.extname(), а потім шукаєте, в якому масиві воно міститься. Якщо розширення не знайдено в жодній категорії, файл потрапляє в Other.

Перед копіюванням файлів створіть усі директорії категорій у цільовій директорії. Використовуйте fs.mkdir(path, { recursive: true }) з опцією recursive, яка автоматично створює всі проміжні директорії в шляху, якщо вони не існують. Це важливо, бо інакше спроба створити вкладену директорію призведе до помилки, якщо батьківська директорія ще не існує.

Для копіювання файлів використовуйте два різні підходи в залежності від розміру файлу. Малі файли (менше 10 MB) копіюйте через fs.copyFile(source, target) — це простий та швидкий метод для невеликих обсягів даних.

Великі файли (10 MB і більше) копіюйте через streams з використанням pipeline():

```python
import { pipeline } from 'stream/promises';
import fs from 'fs';

await pipeline(
  fs.createReadStream(sourcePath),
  fs.createWriteStream(targetPath)
);
```

Чому саме так?

Метод fs.copyFile() зручний для малих файлів, але він завантажує весь файл у пам'ять перед копіюванням. Для великого відеофайлу на 500 MB це означає витрату 500 MB оперативної пам'яті. Streams вирішують цю проблему, обробляючи файл невеликими порціями.

Функція pipeline() з модуля stream/promises автоматично обробляє backpressure та помилки, що робить її найкращим вибором для копіювання великих файлів.

Якщо файл з такою назвою вже існує в цільовій директорії, не перезаписуйте його. Замість цього додайте до назви суфікс із числом: file.pdf → file(1).pdf → file(2).pdf і так далі, як це робить операційна система. Для цього перевіряйте існування файлу через fs.access() або fs.stat(), і якщо файл існує, генеруйте нову назву з наступним порядковим номером.

Створіть клас `Organizer extends EventEmitter` з подіями `'copy-start'` (коли починається копіювання файлу), `'copy-complete'` (коли файл успішно скопійовано) та `'copy-error'` (якщо сталася помилка). Це дозволить показувати прогрес-бар та оновлювати статистику в реальному часі.

### Команда cleanup для видалення старих файлів

Команди викликаються наступним чином.

Щоб показати список старих файлів без видалення (dry run mode)
```
node file-organizer.js cleanup /path/to/directory --older-than 90
```
Та показати список та автоматично видалити з підтвердженням
```
node file-organizer.js cleanup /path/to/directory --older-than 90 --confirm
```

Команда `cleanup` знаходить файли старіші за вказану кількість днів та опціонально видаляє їх. Вік файлу визначається за датою останньої модифікації. Якщо команда викликана без прапорця `--confirm`, вона тільки показує список файлів, які можна видалити, але не видаляє їх фактично (це називається "dry run mode").

Якщо прапорець `--confirm` присутній, файли будуть видалені після показу списку.

Така поведінка робить команду більш безпечною. Користувач спочатку може переглянути, що саме буде видалено, і тільки потім виконати фактичне видалення.

Це запобігає випадковій втраті важливих даних.

#### Технічна реалізація

Для парсингу аргументів командного рядка використовуйте process.argv. Структура process.argv виглядає так: перший елемент - шлях до Node.js, другий - шлях до вашого скрипту, третій і далі - аргументи, передані користувачем. Отже, якщо команда викликана як node file-organizer.js cleanup /path --older-than 90 --confirm, то process.argv[2] буде 'cleanup', process.argv[3] буде '/path', а далі йдуть прапорці.

Для кожного файлу отримайте статистику через fs.stat(filePath). З об'єкта статистики візьміть властивість mtime (modification time) — це об'єкт Date, що вказує на час останньої модифікації файлу.

Обчисліть вік файлу в днях:
```js
const stats = await fs.stat(filePath);
const fileAge = Date.now() - stats.mtime.getTime();
const daysOld = fileAge / (1000 * 60 * 60 * 24);

if (daysOld > threshold) {
  // додати файл до списку на видалення
}
```
Тут Date.now() повертає поточний час у мілісекундах з 1 січня 1970 року, а stats.mtime.getTime() повертає час модифікації файлу в тому ж форматі. Різниця дає вік файлу в мілісекундах. Ділимо на 1000 (секунди), потім на 60 (хвилини), потім на 60 (години), потім на 24 (дні), щоб отримати вік у днях.

Якщо прапорець --confirm відсутній, тільки виведіть список файлів та підсумкову статистику, після чого покажіть повідомлення про dry run mode. Якщо прапорець --confirm присутній, виконайте фактичне видалення через fs.unlink(filePath) для кожного файлу зі списку.

Створіть клас `Cleanup extends EventEmitter` з подіями `'file-found'` (коли знайдено файл для видалення), `'file-deleted'` (коли файл успішно видалено) та `'cleanup-complete'` (коли всі операції завершено). Це дозволить показувати прогрес видалення та оновлювати лічильник оброблених файлів.

## Структура проєкту

Організуйте ваш код у логічну структуру файлів та директорій. Кореневою директорією проєкту буде file-organizer/, де зберігатимуться всі файли застосунку.

У корені проєкту мають бути файли package.json (з конфігурацією проєкту та залежностями), .gitignore (для виключення зайвих файлів з Git), README.md (з документацією) та file-organizer.js (точка входу в програму).

Створіть піддиректорію lib/, де зберігатимуться окремі модулі для кожної команди. Файл lib/scanner.js містить реалізацію команди scan, lib/duplicates.js — команди duplicates, lib/organizer.js — команди organize, а lib/cleanup.js — команди cleanup.

Повна структура виглядає так:
```
file-organizer/
├── package.json
├── .gitignore
├── README.md
├── file-organizer.js
└── lib/
    ├── scanner.js
    ├── duplicates.js
    ├── organizer.js
    └── cleanup.js
```
Файл package.json має містити коректну структуру з полями name, version та обов'язково "type": "module", щоб увімкнути підтримку ES Modules (import/export замість require). Також додайте npm scripts для зручного виклику команд:
```js
{
  "name": "file-organizer",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "scan": "node file-organizer.js scan",
    "duplicates": "node file-organizer.js duplicates",
    "organize": "node file-organizer.js organize",
    "cleanup": "node file-organizer.js cleanup"
  }
}
```
Тепер замість довгої команди `node file-organizer.js scan /path` можна викликати `npm run scan -- /path`. Подвійний дефіс -- передає всі наступні аргументи безпосередньо в скрипт.
